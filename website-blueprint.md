## 1. TECH STACK & ARCHITECTURE

### Core Framework
- Next.js 15+ with App Router — Server Components default, file-system routing, streaming SSR
- React 19 — Server Components, Suspense boundaries
- TypeScript — Strict mode, full type safety across every file

### Styling & Animation
- Tailwind CSS 4 — Utility-first, custom design tokens defined in tailwind.config.ts
- Framer Motion — Page transitions, scroll-triggered reveals, staggered entry animations, parallax, hover micro-interactions
- CSS Custom Properties — Theme tokens for dark backgrounds, neon/electric accent glows, spacing scale, border radii

### Internationalization
- next-intl — App Router native, ~2KB client bundle, type-safe translation keys, Server Component support
- Locale routing via [locale] dynamic segment in the app directory
- Middleware handles automatic locale detection and redirect
- Romanian (ro) = default locale, English (en) = secondary
- All URLs are Romanian-first: /ro/industrii/clinici-dentare and /en/industries/dental-clinics

### Data Visualization
- Recharts — For ROI calculators, before/after comparisons, revenue impact bars, analytics mock-ups
- Custom SVG components — Animated infographics, process flow diagrams, stat counters with counting animation
- Chart.js as backup for more complex interactive dashboards

### Forms & Interaction
- React Hook Form + Zod — Contact forms, demo request forms, lead capture with validation
- Calendly embed or custom booking component — For scheduling live demos
- Toast notifications for form submission feedback

### Deployment
- Vercel — Edge network, automatic HTTPS, preview deploys per branch, built-in Web Analytics
- Custom .ro domain for maximum Romanian geo SEO signal
- Automatic sitemap generation via next-sitemap

## 5. DESIGN SYSTEM & VISUAL IDENTITY

### Aesthetic Direction: DARK TECH-FORWARD MINIMALISM
Deep dark backgrounds. Electric accent glows. Clean geometric layouts. Data-driven visuals. High-end SaaS meets restrained cyberpunk. NOT generic corporate blue. NOT startup pastel.

### Color Palette
```css
:root {
  --bg-primary: #0A0A0F;
  --bg-secondary: #111118;
  --bg-card: #16161F;
  --bg-elevated: #1C1C28;
  --accent-primary: #00E5CC;
  --accent-secondary: #6366F1;
  --accent-glow: rgba(0, 229, 204, 0.15);
  --text-primary: #F0F0F5;
  --text-secondary: #8B8B9E;
  --text-accent: #00E5CC;
  --border-subtle: rgba(255, 255, 255, 0.06);
  --border-glow: rgba(0, 229, 204, 0.3);
  --positive: #22C55E;
  --negative: #EF4444;
  --neutral: #6366F1;
}
```

### Typography
- Display/Headings: Satoshi (self-hosted) — bold, geometric, modern tech feel
- Body text: General Sans or DM Sans — clean, readable, warm
- Monospace accents: JetBrains Mono — stat numbers, pricing figures, data labels (dashboard feel)
- Scale: Tailwind default with custom overrides for hero (clamp-based fluid sizing)

### Animation Principles
- Page load: Staggered fade-up with 50ms delay between elements
- Scroll reveals: Elements fade up + slight scale (0.95 to 1) on viewport entry, triggered once
- Hover: Cards get border-glow effect, buttons scale 1.02 with shadow lift
- Charts: Bars grow from 0 to value on scroll-into-view, numbers count up in real time
- Page transitions: Crossfade 200ms with Framer Motion AnimatePresence
- Parallax: Subtle on hero background shapes (10-15px), not distracting

### Card Styles
- Feature cards: Dark surface, 1px subtle border, hover becomes glow border with box-shadow
- Stat cards: Dark surface, large monospace number on top, small label below, number animates
- Pricing cards: Center card is taller, has accent border, "Recomandat" badge
- Industry cards: Icon at top, industry name, one-line description, arrow, grid of 2x5

### Responsive Breakpoints
- Mobile: 320px-767px — single column, hamburger, stacked
- Tablet: 768px-1023px — 2-column grids, condensed nav
- Desktop: 1024px-1439px — full layout, mega dropdown
- Large: 1440px+ — max-width 1280px container, centered

## 10. CONTENT WRITING RULES

### Voice
- Sharp Romanian business consultant, not chatbot
- Short punchy sentences mixed with longer explanatory ones
- Romanian idioms: "Bani aruncați pe fereastră" not "fonduri pierdute inutil"
- Rhetorical questions: "Câți clienți ai pierdut luna asta?"
- Incomplete sentences for emphasis: "Fără răspuns. Fără programare. Fără client."

### AVOID (AI Tells)
- "leverage", "utilize", "harness", "cutting-edge", "game-changer", "seamless", "robust"
- "În peisajul actual al..." / "Este important de menționat că..."
- "Acest lucru permite..." / "În concluzie" (unless actual conclusion)
- "Soluții inovatoare" — replace with specifics
- 3+ adjectives before a noun

### USE
- "Uite care-i treaba:" / "Hai să punem pe hârtie" / "Fă un calcul rapid"
- "Gândește-te la asta:" / "Nu e complicat." / "Testează-ne. Dacă nu merge, pleci."
- "Fără vrăjeală." / Industry-specific slang

### Formatting
- Max 3-4 sentences per paragraph
- Numbers not words: "3 zile" not "trei zile"
- € symbol in body text
- Bold sparingly — key stats or first occurrence of important claim
- Visual break between every section

## 11. RUFLO — MULTI-AGENT WORKFLOW FOR BUILDING THIS WEBSITE

### What Ruflo Actually Is

Ruflo (github.com/ruvnet/ruflo) is a multi-agent orchestration platform built for Claude Code. Instead of one single agent doing everything — design decisions, writing code, handling content, testing — Ruflo lets you deploy a swarm of specialized agents that work in parallel on different parts of the project. Each agent has a defined role (coder, architect, reviewer, tester) and they coordinate through shared memory so the work stays consistent across the entire codebase.

The analogy: instead of one person building the house, you have an architect drawing plans, a plumber doing pipes, an electrician doing wiring, and a foreman keeping it all connected. Ruflo is the foreman.

### Why Ruflo Matters for This Project

This website has 15+ pages, 30+ components, 2 languages, strict design requirements, SEO schema on every page, and interactive charts. A single Claude Code session would hit context window limits before finishing half of it. Code quality degrades as context fills up. Decisions from page 1 get forgotten by page 5.

With Ruflo, each agent works in its own focused context. The Component-Builder only thinks about components. The Page-Builder only thinks about pages. The shared memory keeps everyone using the same design tokens, naming conventions, and architectural decisions from this blueprint.

### Step 1: Install and Initialize

```bash
# Make sure you have Claude Code installed and working first
# Then install Ruflo
npm install -g ruflo@latest

# Go to your project folder
cd agency-website

# Initialize Ruflo in the project
npx ruflo@latest init
```

This creates two things in your project:
- A `CLAUDE.md` file — instructions that tell the agent system how to behave in this project
- A `.swarm/` directory — where memory, agent configs, and coordination data live

After init, Ruflo hooks into Claude Code. When you run Claude Code in this project folder, it automatically uses the orchestration layer.

### Step 2: The Agent Types You Need

Ruflo ships with pre-built agent types. For this website build, the relevant ones are:

- **architect** — Plans structure, creates scaffolding, makes sure file organization follows the blueprint
- **coder** — Writes React/Next.js components, pages, styles. Does the implementation
- **reviewer** — Reviews code from coder agents. Catches TypeScript issues, consistency problems, accessibility gaps
- **tester** — Writes tests, verifies pages render, checks i18n switches correctly
- **coordinator** — The boss agent. Reads the high-level task, breaks it into subtasks, assigns them to specialist agents, tracks progress

You can also define custom agents (like a "content-writer" agent that fills in translation JSON files, or an "seo-auditor" that checks metadata).

### Step 3: Start the Swarm

```bash
# Start the daemon (runs in background, manages coordination)
npx ruflo@latest daemon start

# Initialize a hierarchical swarm (coordinator at top, specialists below)
npx ruflo@latest swarm init --topology hierarchical

# Spawn the coordinator agent
npx ruflo@latest agent spawn --type coordinator --name "Website-Lead"

# Spawn specialist agents
npx ruflo@latest agent spawn --type architect --name "Structure-Architect"
npx ruflo@latest agent spawn --type coder --name "Component-Builder"
npx ruflo@latest agent spawn --type coder --name "Page-Builder"
npx ruflo@latest agent spawn --type coder --name "Industry-Pages-Builder"
npx ruflo@latest agent spawn --type reviewer --name "Code-Reviewer"
```

### Step 4: Feed the Blueprint

Give the coordinator this MD file as the project spec:

```bash
npx ruflo@latest task create \
  "Build the agency website following website-blueprint.md. \
   Phase 1: Scaffold Next.js 15 project with App Router, TypeScript, Tailwind, next-intl. \
   Phase 2: Build component library (all components from the blueprint). \
   Phase 3: Implement landing page with all 10 sections. \
   Phase 4: Build core pages (pricing, how-it-works, about, contact, FAQ). \
   Phase 5: Build all 10 industry pages from the template. \
   Phase 6: SEO metadata, schema markup, OG images. \
   Phase 7: Animations, responsive testing, polish." \
  --assign "Website-Lead"
```

The coordinator breaks this into subtasks and delegates:
- Architecture agent scaffolds the project, creates configs, sets up routing and i18n
- Component-Builder creates all reusable UI pieces (Button, Card, Hero, PricingTable, etc.)
- Page-Builder does landing page and core pages
- Industry-Pages-Builder creates all 10 industry pages in parallel
- Code-Reviewer checks each deliverable

### Step 5: Parallel Execution

The real speed gain. Multiple agents build different pages simultaneously:

```bash
# One agent handles the 10 industry pages
npx ruflo@latest task create \
  "Build all 10 industry pages following the template in website-blueprint.md" \
  --assign "Industry-Pages-Builder"

# Another handles core pages at the same time
npx ruflo@latest task create \
  "Build pricing, how-it-works, about, contact, FAQ pages" \
  --assign "Page-Builder"
```

Both work at the same time. Shared memory keeps them aligned on design tokens, component APIs, and structure.

### Step 6: MCP Integration

Ruflo can be added as an MCP server inside Claude Code, so Claude Code itself can use Ruflo tools directly from the conversation:

```bash
# Add Ruflo as MCP server to Claude Code
claude mcp add ruflo -- npx -y ruflo@latest mcp start

# Verify
claude mcp list
```

Now when you chat with Claude Code in your terminal, it can spawn agents, create tasks, check swarm progress — all from the conversation.


### Why This Beats Working With One Agent

- Each agent has focused context — no forgetting, no drift
- Shared memory keeps consistency across all agents
- Parallel execution: 3-4 agents at once instead of sequential
- Reviewer catches what coders miss
- Coordinator maintains the big picture against this blueprint
- For a 15+ page project with 30+ components and 2 languages, this is the right tool
